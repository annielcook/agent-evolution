From Prompt to Platform

How civic innovators can evolve one-off LLM chats into testable, scalable agents.

Public sector leaders are increasingly experimenting with AI — but many start with simple chats and stop there.

Agents are systems that take inputs, follow structured logic, and deliver outputs — like a teammate that can think, decide, and act on your behalf.

The good news? You can build your first agent today using tools you already know — like ChatGPT, Zapier, and a little creative prompting.

One Civic Agent, Evolving in 5 Steps

We wanted to build an agent that helps public innovators test their civic ideas faster and with less waste. What began as a single prompt became a small system of connected tools — a mini R&D teammate anyone can activate.

Step 1: LLM Prompting

The simplest way to start is just by asking an LLM a question. For example:

How can I improve recycling rates in my city?

That gets you an answer. But it doesn’t use the LLM to its full potential.

Instead, you can structure your prompt to simulate a mini-agent: giving it steps, judgment, and a process to follow. Here's how that looks:

You are a Civic Innovation AI Agent designed to help public entrepreneurs explore and test ideas. Your job is to take a civic problem, generate thoughtful solutions, and then turn those into testable MVPs — all with minimal waste and maximum learning. When given a civic problem, follow this process: Step 1: Solution Generation Propose 3–5 diverse potential solutions. Vary by cost, ambition, and time horizon. Label each clearly. Step 2: MVP Formulation for Each Solution Suggest one or two ways to test each solution using minimal time, money, and resources. Focus on learning, not perfection. Step 3: “Leaner Than That” Cycle For each MVP, suggest an even leaner version that could produce similar learning with less effort. Step 4: Persona-Based Picks Choose one MVP that Eric Ries would likely pick (with brief reasoning), and one Santiago Garces might pick.

Pros

  • Extremely easy to set up — no new tools required
  • Quick to change and rerun — just tweak the prompt
  • Can run over and over — flexible for repeated needs
  • Clear visibility — the steps and thinking are transparent
  • Fits into daily workflow — no need to leave ChatGPT

Cons

  • Tedious to share — others must copy-paste full prompts
  • Hard to manage updates — changes live inside chat memory
  • Lives in LLM memory — no external version control
  • Fragile across sessions — chat context may be lost

To make the agent more useful — easier to share, easier to update, and more reliable — the next step was to give it a simple structure beyond a chat window by wrapping it in a website.
