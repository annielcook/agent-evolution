Step 1: LLM Prompting

The simplest way to start is just by asking an LLM a question. For example:

How can I improve recycling rates in my city?

That gets you an answer. But it doesn't use the LLM to its full potential.

Instead, you can structure your prompt to simulate a mini-agent: giving it steps, judgment, and a process to follow. Here's how that looks:

You are a Civic Innovation AI Agent designed to help public entrepreneurs explore and test ideas. Your job is to take a civic problem, generate thoughtful solutions, and then turn those into testable MVPs ‚Äî all with minimal waste and maximum learning. When given a civic problem, follow this process: Step 1: Solution Generation Propose 3‚Äì5 diverse potential solutions. Vary by cost, ambition, and time horizon. Label each clearly. Step 2: MVP Formulation for Each Solution Suggest one or two ways to test each solution using minimal time, money, and resources. Focus on learning, not perfection. Step 3: "Leaner Than That" Cycle For each MVP, suggest an even leaner version that could produce similar learning with less effort. Step 4: Persona-Based Picks Choose one MVP that Eric Ries would likely pick (with brief reasoning), and one Santiago Garces might pick.

  • Extremely easy to set up ‚Äî no new tools required
  • Quick to change and rerun ‚Äî just tweak the prompt
  • Can run over and over ‚Äî flexible for repeated needs
  • Clear visibility ‚Äî the steps and thinking are transparent
  • Fits into daily workflow ‚Äî no need to leave ChatGPT

Cons

  • Tedious to share ‚Äî others must copy-paste full prompts
  • Hard to manage updates ‚Äî changes live inside chat memory

To make the agent more useful ‚Äî easier to share, easier to update, and more reliable ‚Äî the next step was to give it a simple structure beyond a chat window by wrapping it in a website.

Step 2: Wrapping the Agent in a Website

Once you've crafted a useful, structured prompt, the next step is making it reusable and shareable. One of the easiest ways to do that is by wrapping it in a simple web app.

With tools like Replit, Vercel, Glitch, or even ChatGPT's code interpreter, you can scaffold a working site with:

  • Your prompt logic
  • A form for user input
  • An API key for your preferred LLM

You don't need to be a web developer. Just drop in your logic, plug in your API key, and you have a tool anyone can use.

Step 3: Automating with Workflows

Now that your agent lives in a website, you can take it a step further ‚Äî and have it run automatically in the background. This is where workflows come in.

Using tools like Zapier, Make, n8n, or Pipedream, you can trigger your agent to run:

  • When someone submits a form
  • When a new row is added to a spreadsheet
  • On a schedule or from an email

And once it runs, the results can be sent anywhere ‚Äî emailed, logged, shared in Slack, or saved in a dashboard.
